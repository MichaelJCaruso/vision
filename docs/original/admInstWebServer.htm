Vision Pre-Installation: Web Server Information

Topics Covered

  • Overview
  • Web Administrative Tasks
  • Tips

Overview

Vision Applications are delivered to users via a Web-based browser running on an internal intranet.

  • Web documents provided in the installation are HTML version 3.2 compliant. Components utilized include HTML Forms, HTML Tables, and Java Applets.

  • The CGI application provided in the installation is CGI version 1.1 compliant.

  • Known Web servers that have hosted these components in the past include:

      NCSA HTTPd server
      Microsoft Peer-Web-Services 2.0 & 3.0
      Microsoft IIS 2.0 & 3.0
      Oracle Web Server
      Netscape SuiteSpot 3.5 Web Server

  • Any Web browser capable of HTML 3.2 should be compatible with the supplied applications and documents including: Netscape Navigator 3.1 & 4.0 and Microsoft Explorer 3.x & 4.0.

  • Some applications utilize Java class libraries from Visual Mining Inc. Prior to the applet running in your browser window you will receive a reminder to purchase the NetCharts server license. The license can be obtained from Visual Mining.

Web Administrative Tasks

The installation of the Vision Web Components requires some administrative tasks to be completed on the Web Server such as; Configuration of new directory aliases, and Modification of Server Environment Variables. Familiarity with the Web Server's Administration process is required.

Tips

There are many different Web servers available, each having special setup requirements. The following summarizes the issues that have arisen with different Web servers:

Microsoft Peer-Web-Services 2.0 & 3.0

  • The cgi-bin directory is not created automatically. It should be created and configured to be accessible from the root of the web server (i.e., http://www.mycompany.com/cgi-bin/).

Netscape SuiteSpot 3.5 Web Server

  • The cgi-bin directory is not created automatically. It should be created and configured to be accessible from the root of the web server (i.e., http://www.mycompany.com/cgi-bin/).
  • The environment variable VQueryConfig will need to be added to the obj.conf file.
  • The following example shows an obj.conf file after the Vision Web components have been placed on the Web Server.
      # Netscape Communications Corporation - obj.conf
# You can edit this file, but comments and formatting changes
# might be lost when the admin server makes changes.

# Use only forward slashes in pathnames--backslashes can cause
# problems. See the documentation for more information.

Init fn="flex-init"
access="C:/Netscape/SuiteSpot/https-visionWeb/logs/access"
format.access="%Ses->client.ip% - %Req->vars.auth-user% [%SYSDATE%]
\"%Req->reqpb.clf-request%\" %Req->srvhdrs.clf-status%
%Req->srvhdrs.content-length%"
Init fn="load-types" mime-types="mime.types"
Init fn="load-modules"
shlib="C:/Netscape/SuiteSpot/plugins/content_mgr/bin/content_mgr.dll"
...
VQueryConfig="c:/netscape/suitespot/vision/cgi-misc/vquery.cfg"

Object name="default"
NameTrans fn="pfx2dir" from="/ns-icons"
dir="C:/Netscape/SuiteSpot/ns-icons"
NameTrans fn="pfx2dir" from="/mc-icons"
dir="C:/Netscape/SuiteSpot/ns-icons"
NameTrans fn="pfx2dir" from="/cgi-bin"
dir="C:/Netscape/SuiteSpot/VISION/cgi-misc" name="cgi"
NameTrans fn="pfx2dir" from="/vision" dir="c:/netscape/suitespot/vision"
...
