Overview

The core Vision system comes with a set of classes already defined. These classes include an initial set of methods which can be modified and extended by the user. Vision's built-in class hierarchy is discussed in detail in the general Vision documentation. The Portfolio Management Application Layer defines classes, protocol, and starter applications that address the needs of the portfolio management function. This layer is entirely implemented using the Vision language and can be readily tailored to meet the specific needs of your organization.

Note that the actual entities created for the core investment classes will be dependent on your installation's requirements. Many of the named entities used in the examples in these documents are included for illustration purposes only and may not exist in your environment. You can create them using the techniques provided or substitute entity identifiers that are defined for you organization.

Installation

The Vision Run-Time Environment provides the tools needed to build and maintain a Vision database designed to deliver Web-based applications to users via an internal intranet. Basic capabilities provided in this environment include a starter suite of web-based applications, initial templates for loading portfolio, security, and supporting information, and basic tools for committing data to the database.

The document, Portfolio Management Application: Installation describes the implementation process.

Headstart Applications and Analyst Tools

The Portfolio Management Application Layer includes a starter suite of web-based applications. These applications are accessed with a web browser via an organization's intranet. The Headstart applications are described in the document, Portfolio Management Application: Headstart Application.

In addition, analysts can interact with Vision using Excel Add-ins designed to extract data from Vision and temporarily define and modify data that is used in conjunction with browser applications. These analyst tools are described in the document, Portfolio Management Application: Analyst Tools.

Uploading Data

Standardized interfaces for loading data from external sources are provided as part of this Application Layer. An Excel interface is available for making temporary modifications to the database. These changes will be reflected in an analyst's private versions of browser reports, but will not be saved permanently. Tools for committing permanent changes to the database are also provided.

The pre-defined data feeds are described in the document, Portfolio Management Application: Feeds.

Database Updates and Administration

A collection of database administration tools are available for loading data into the database, defining additional database schema, and performing various database maintenance functions.

A single user is designated as the Vision database administrator and is given access to these tools which are described in the document, Portfolio Management Application: Administration .

Getting Started

Once you have installed your software, reviewed the data feed requirements, and created an initial set of feeds, you are ready to start loading and using your data. Samples of this process are described in the document, Portfolio Management Application: Getting Started.

Next Steps

Over time, you will probably want to add new reports and expand the set of internal and external data feeds you incorporate. You will be able to do many of these enhancements on your own using the techniques you learned during the installation process. At this point, you may want to learn more about the Vision language so you can modify and create additional applications. In addition, you may want to familiarize yourself with the production tools that have been defined to help manage your Vision environment.

More information about "next steps" is available.

Core Classes

The Portfolio Management Application Layer defines a number of core classes used to manage investment data. Core entity classes include Company, Security, Industry, and Account. Additional information is managed by subclasses of DataRecord and LinkRecord including PriceRecord, DivRecord, and Holding.

These classes are described in detail in the document, Portfolio Management Application: Classes.

Supplemental Core Classes

The Portfolio Management Application Layer defines a number of supplemental core classes that are available as part of the Headstart Application Kit. Supplemental entity classes include Analyst and Manager. Additional information is managed by subclasses of DataRecord including fundamental company data and country economics.

These classes are described in detail in the document, Portfolio Management Application: Supplemental Classes.

Special Issues

Special structures have been designed to manage holdings, dividend, price, and adjustment data in the Vision database. These structures have been created to correctly handle split and currency adjustments automatically. In addition, these structures have been designed to facilitate future reorganizations as the database grows, so that applications that use this data are sheltered from structural changes.

The document, Portfolio Management Application: Issues describes these issues in more detail.
