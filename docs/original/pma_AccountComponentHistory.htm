Application: AccountComponentHistory

 

Summary:

The AccountComponentHistory application is used to display the component members comprising an AggregateAccount or a CompositeAccount.

Inputs:

Position Form Id Description 
1  id  Id of aggregate or composite account 

Analysis:

  • The id must reference an existing AggAccount or CompositeAccount instance.
  • For aggregate accounts, two reports are displayed. For each date on which portfolio membership in the aggregate changes, the date, and the portfolios in the aggregate as of that date are displayed. For each date on which holdings have been created, the market value of the aggregates and its member portfolios are displayed.
  • For composite accounts, one report is displayed. For each date on which the membership in the composite changes, the date, and the accounts in the composite and their weights in the composited are displayed.

Key Data Feeds:

  • PortfolioAggregates
  • CompositeAccountMembers

Related Reports:

  • AccountProfile
  • AccountHoldings

     Interface ApplicationWS runWith: "AccountComponentHistory", "Agg901" ;
